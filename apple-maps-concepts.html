Apple Maps Concepts

Overview

Project Goal

Adding features to Apple Maps that I wish it had in a realistic manner.

Objectives

Add useful features
Build on existing design language
Integrate design seamlessly
Practice mobile design

Background

Being an iOS user and living in Berlin, I use Apple Maps pretty much on a daily basis, especially for getting directions when using public transport or driving. Features like live location of buses, trains, etc. and a beautiful, highly detailed map make the user experience great, but over time I have had some moments where I wished for some additional information.

I started collecting my ideas for new features and thought about how it could look if Apple were to implement them in Maps.

The first feature I wish Maps had, is being able to see the platforms trains stop at to make it easier to find the one you're looking for. Then, within train lines, there could also be distinctions between which direction a specific train is heading, as often there are different end stations on the same line. This would be even more useful for people who don't care about the end station, but just want to know if the next train is going in their direction.

Next, I believe that a live departure board for airports could be a simple, yet useful, feature. This could even be integrated with Apple's already existing “flight info” feature in iMessage, by showing friends' and family member's flights in Maps.

Lastly: please just give me a little bit more info when driving, Apple Maps. Of course it would be great to have all important road signs showing up on screen to make sure they haven't been overlooked, but I know that would be hard to implement. However, at least showing the actual speed limit and parking restrictions (taking time of day and day of week into account for both) could be done and would be immensely useful. I can't tell you how often Maps has shown me the wrong speed limit because it doesn't take the extra sign, specifying the time range for a speed limit, into account.

Disclaimer: My Apple Maps Concepts are in German (for example 'Gl.' is the German abbreviation for 'platform').

Platform Information

Departure Board

This is currently the kind of departure board that can be seen when clicking on any station in Apple Maps. It's pretty nice, but with a few tweaks it could show even more valuable information to the user.

It could, for example, display platform numbers in the middle, giving the user this information at a glance when scrolling through the list. By removing separators between trains heading in the same direction and grouping them, we could make it so much easier for users who only care about going in a certain direction on a train line and not a specific end stop.

Now, we could also add a filter at the top to sort the departure board by platforms. This could help a user to, for example, find all trains departing on the platform he's standing on, because sometimes it doesn't even matter what train line the next train is, but only which direction it's going.

Detailed Info Screen

When clicking on a train, the user is brought to this detailed info screen in the current state. This could also be updated with some platform information.

In the scrollable container at the top, I grouped the trains heading in the same direction again, using a separator to achieve the same effect as in the departure board. Below that, the departures of trains going to the specified end station is shown with the addition of platforms numbers. This is important because, sometimes, these can vary even on regular services (especially on regional trains here in Germany). Lastly, I added platform information for all stations in the route preview at the bottom.

Airport Overhaul

Departure Board

This is currently the kind of screen that can be seen when clicking on airports on Apple Maps. Even though there is already so much transport-related info available on Maps, from ferries to cable cars, the user is left on his own to find any info when he's at the airport. How about we add a departure board to this screen?

Much better. The user can now look at current departures (Abflüge) and arrivals (Ankünfte) in an interface resembling the ones he already knows from places like train stations. To view more flights, the user can click on 'View all' ('Alle anzeigen'). Below you can see how I imagine that screen could look.

Here the user can view all departures and arrivals of the day. There is a prominent distinction between past and future flights using the current time as a separator. When designing this, I wanted to stop here at first, but then I remembered that Apple already has a design for viewing details of a specific flight. Weirdly, however, this is only available when sending a flight number through iMessage. So why not integrate that screen in Maps?

Messages Integration

This is how it could look when clicking on a specific flight. I simply took Apple's already existing design from Messages and integrated it into Apple Maps. Piece of cake! Now why not also show flights that you've received in Messages from friends and family in the airport screen?

I designed this card to resemble the departure board, but cut some less important information and replaced it with more relevant details.

Now, when a user clicks on an airport that matches a flight that he's received in Messages, he can see that flight right away at the top and know if it'll be on time, for example. With these design changes, the under-used and not well-known flight info screen from Messages can be put to much better use, I think.

Road Signage

Displaying more Road Signs

This idea is pretty simple. Currently Apple Maps only displays speed limits like here, but sometimes that's not enough.

My revised version would show more accurate speed limits, taking the time range for a speed limit (if specified like in the example above) into account. Additionally, it could be helpful to know the parking restrictions for the street you're currently on since these signs can be easily overlooked. Thus, I placed this sign less prominently, but still visible, in the bottom right corner.

Project Takeaways

Working with existing components

Before being able to really start working on my own ideas, I first had to study and meticulously copy Apple's existing components and design language to be able to convincingly create designs that looked like they could be from Apple. This took some time, but, in the end, was worth it.

Integrating new features into old designs

Not only did I need to emulate Apple's design language in my own components, but also integrate them into already existing screens seamlessly. Since most of my changes were in the details, many parts of the design stayed the same. This required me to make new spaces in between the existing elements without compromising the design and flow of a screen.

Passion projects

I started this project completely out of my own passion for design and Apple Maps. I came to realize how important it is to have projects besides what you're doing for work or your studies to give yourself the chance to do whatever you want, without any pressure, once in a while.
