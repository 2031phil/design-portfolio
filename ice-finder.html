ICE Finder

Overview

Project Goal

Creating a simple tool to look up ICE train types used on german high speed lines.

Objectives

Make data easier to find
Fun to use, but still simple
Smooth, reliable animations
Deploy to production

Background

I’ve never considered myself a train nerd, but I have recently become more interested in the high speed ICE (Intercity-Express) system here in Germany. One thing that’s been bugging me, however, was that the Deutsche Bahn app doesn’t show the user which ICE train generation is used on which line before the day of the travel (and even then that information is hidden very well).

The train generation you travel with can make a big difference though, as comfort and travel speed vary widely between them. So out of this use case I started looking on the internet for tools to get this information, only to find that, while they exist, they are not easy or quick to use and they don’t provide pictures or stats of the trains themselves either, so if you don’t know the difference between an ICE 3 and ICE 3 neo, that’s an additional Google search.

I then started designing my own tool, keeping it as simple as it needed to be since there wasn’t much it needed to do and I was mainly designing it for myself to use. The primary color had to be the DB red of course. Then there needed to be an input field in the middle and a title above. Translated the title says “What ICE-Type am I traveling with?”.

Now to the centerpiece of this application: The data. Where do I get it from? Sadly not the DB itself. They do not provide an API that grants access to this data. So now is the time that I have to confess that I’ve had to resort to scraping the data from another website, which (I don’t know how) has access to this information. Because of that I’ve decided to prevent search engines from indexing my site and have been using it just for myself, not publishing it.

Design Process

Train Details Screen

Search Results

After typing in a train number and hitting Enter, the tool displays a picture of the corresponding train and its name. When clicking on the chevron it reveals route- and train-information. If there is more than one service on a given train number, the tool displays a selector screen with all of the options.

There are currently 7 ICE types in service. From oldest to newest they are ranked as follows: ICE 1 (1991), ICE 2 (1996), ICE T (1999), ICE 3 (2000), ICE 4 (2017), ICE 3neo (2022), ICE L (2025).

Search Result

Train Details

  • When opening the details, the ICE type moves to the top and stays there, even when scrolling

  • The site automatically scrolls to the first container smoothly
  • It also scrolls back to the top when the details are collapsed again

Development Process

Version 1

When I started this project I was only familiar with simple HTML, CSS and JavaScript so that’s what I coded the site with. The animations were all done with CSS and JS. The backend consisted of a node server, which I was able to code with the help of ChatGPT and Claude.

This project was also the first where I actively focused on good error handling. I implemented a function that could receive error messages and display them for a couple of seconds to the user beneath the search bar. It can distinguish between invalid user input, empty searches, server errors, connection issues, and nonexistent train numbers.

Version 1 in VS Code

Version 2

Some months later I started learning React and changed the hosting of my projects to Vercel. Thus, the project had to be converted to nextjs with most of the heavy lifting done once again by ChatGPT and Claude. However, I forgot to specify in my prompt to not convert to TypeScript and module based CSS (which I hadn’t worked with yet), so that caused some confusion for me at the beginning. In the end I was too lazy to regenerate everything again and took it as an opportunity to expand my knowledge.

After the refactoring it mostly worked, but the animations were not transferred one to one. This took me quite some time to figure out and fix, but in the end I got everything to work. The whole process also gave me the opportunity to improve the error popup design and I discovered new edge cases which broke some animations again.
